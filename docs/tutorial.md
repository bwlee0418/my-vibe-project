# 비개발자를 위한 바이브 코딩 튜토리얼

이 문서는 프로그래밍 경험이 없는 분이 AI(클로드)와 자연어 대화만으로 소프트웨어 프로젝트를 기획하고 설계하는 전체 과정을 정리한 튜토리얼입니다.

## 바이브 코딩이란?

바이브 코딩은 코드를 직접 작성하지 않고, AI에게 자연어(일상 언어)로 지시하여 소프트웨어를 만드는 방식입니다. "이런 기능이 필요해"라고 말하면 AI가 코드를 작성해주는 것이죠.

## 워크숍에서 진행한 단계

### 1단계: 문제 정의 (브레인스토밍)

내 업무에서 가장 시간이 많이 걸리는 반복 업무를 찾고, AI로 해결할 수 있는 방법을 탐색했습니다.

**사용한 프롬프트:**
```
저는 CX팀 팀장을 담당하고 있습니다. 매일 반복하는 업무 중에서 가장 시간이 많이 걸리는 것은
회의 내용 정리와 매주, 매월 보고자료를 작성하는 것입니다.
이 업무를 AI로 자동화하거나 효율화할 수 있는 방법을 함께 고민해주세요.
가능한 해결 방안을 3가지 이상 제안해주시고, 각 방안의 장단점도 함께 알려주세요.
```

**결과물:** AI가 3가지 해결 방안(회의록+보고서 통합, 보고서 템플릿 자동화, CX 대시보드)을 제안했고, 각각의 장단점을 비교하여 최적의 방향을 선택했습니다.

**결과 파일:** `ideation.md`

### 2단계: PRD 작성 (심층 인터뷰)

AI가 인터뷰어 역할을 하며 한 가지씩 질문하고, 답변을 바탕으로 제품 요구사항 문서(PRD)를 작성했습니다.

**사용한 프롬프트:**
```
ideation.md 파일을 읽고, 이 아이디어로부터 프로젝트 PRD를 만들기 위한
심층 인터뷰를 진행해주세요. 한 번에 모든 질문을 하지 말고,
한 가지 주제씩 순서대로 물어봐주세요.
```

**인터뷰에서 다룬 주제:**
- 도구의 사용자는 누구인가 → CX팀 전체 + 상급 관리자
- 핵심 기능과 우선순위 → 회의록 자동 정리, 주간 보고서 자동 생성
- 성공 기준 → 시간 절약 + 품질 향상 모두 중요
- 제약 조건 → 워크숍 실습용으로 특별한 제약 없음

**결과 파일:** `PRD.md`

### 3단계: 목업 제작

PRD를 기반으로 실제 동작하는 HTML 목업을 만들었습니다. 단일 HTML 파일로 서버 없이 브라우저에서 바로 확인할 수 있습니다.

**사용한 프롬프트:**
```
PRD.md에 기반해서 동작하는 HTML 목업을 만들어줘.
조건:
- 단일 HTML 파일로 만들어줘 (별도 서버 없이 브라우저에서 바로 열 수 있게)
- Tailwind CSS CDN을 사용해서 깔끔하게 디자인해줘
- 반응형으로 만들어줘
- 샘플 데이터를 넣어서 실제 동작하는 것처럼 보이게 해줘
```

**결과물:** 3개 화면(회의록, 보고서, CX 지표)이 포함된 동작하는 HTML 목업

**결과 파일:** `mockup.html`

### 4단계: 개발 계획 수립 + GitHub 이슈 등록

개발자가 실제로 개발할 수 있도록 기술 스택, 작업 목록, 의존성을 정리하고 GitHub 이슈로 등록했습니다.

**사용한 프롬프트:**
```
이 프로젝트를 개발자들이 실제로 개발할 수 있도록 계획 문서를 작성해줘.
```

```
계획 문서의 각 작업을 gh cli를 사용해서 GitHub 이슈로 등록해줘.
각 이슈에는 배경, 작업 내용, 인수 조건, 의존성이 포함되어야 해.
```

**결과물:** 17개 작업이 우선순위별로 정리된 개발 계획 + 11개 GitHub 이슈

**결과 파일:** `development-plan.md`

### 5단계: 아키텍처 문서화

Mermaid 다이어그램으로 시스템 구조, 컴포넌트 관계, 데이터 흐름을 시각화했습니다.

**사용한 프롬프트:**
```
이 프로젝트의 아키텍처 문서를 생성해줘.
다이어그램은 mermaid로 그려줘. README도 업데이트해줘.
```

**결과물:** 4개의 Mermaid 다이어그램 (시스템 구조, 컴포넌트 관계, 데이터 흐름, 기술 스택)

**결과 파일:** `architecture.md`, `README.md`

### 6단계: 튜토리얼 및 매뉴얼 작성

이 문서(지금 읽고 계신 것)와 비개발자용 Git 매뉴얼을 작성했습니다.

**결과 파일:** `docs/tutorial.md`, `docs/git-manual.md`

## 핵심 프롬프트 패턴

### 패턴 1: 역할 + 맥락 + 출력형식
```
당신은 [역할]입니다. [배경 상황 설명]
[원하는 결과물의 형식]으로 작성해주세요.
```

### 패턴 2: 단계적 질문 (심층 인터뷰)
```
[주제]에 대해 심층 인터뷰를 진행해주세요.
한 번에 한 가지 질문만 하고, 제 답변을 바탕으로 다음 질문을 만들어주세요.
```

### 패턴 3: 검수 루프
```
[결과물]을 만들어주세요.
완성 후 스스로 검토해서 부족한 점을 수정해주세요.
```

### 패턴 4: 구체적 조건 지정
```
다음 조건으로 만들어주세요:
- [조건 1]
- [조건 2]
- [조건 3]
```

## 워크숍 후 실천 항목

1. 오늘 배운 프롬프트 패턴을 내일 업무에 바로 적용해보세요
2. 팀원에게 유용한 프롬프트를 공유하세요
3. 반복 업무 하나를 골라 클로드와 함께 자동화해보세요
4. 더 도전적인 프로젝트에 바이브 코딩을 시도해보세요
